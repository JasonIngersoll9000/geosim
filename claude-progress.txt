### Architecture decisions
- Using bun (not npm/npx) for all package management — npm/npx are Windows binaries
  in WSL2 and cannot execute Linux binaries
- Vitest configured with node environment (not jsdom) for server-side logic tests
- docs/ excluded from TypeScript compilation (reference docs, not compiled code)
- Skill triggers wired via PostToolUse hooks (not memory) so automation is reliable
- features.json tracks pass/fail per feature — only mark passes:true after tests pass
- ESLint argsIgnorePattern: ^_ must be in .eslintrc.json (Vercel build requires it for _-prefixed params)
- prefer-const applies even when mutating object properties via bracket notation — binding must be const
- Concurrent TurnPlan validation must check every concurrent pair (i<j loop), not just vs primary
